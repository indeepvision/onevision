Camera Configuration

The basic configuration sub-menu is used to configure the parameters of the camera controller itself, not the internal features of the camera as a device. In summary, here you can configure how OneVision and the camera will work together.

Friendly name​

The friendly name is the unique name of the camera controller that will be used throughout the proejct. It must be unique, but can be changed at any time.

Enable/disable camera​

This option is used to enable or disable the camera controller. If the controller is disabled, it will not be used in the project and will not connect to the camera device.

Critical​

This option is used to mark the camera controller as critical. If the controller is critical, the software will consider an error if the camera is not connected. For example, runtime execution will stop if connection with the camera device is lost.

When a critical camera is not connected, the camera tab will display a "red" icon next to the camera name. If it is not connected but it is not critical, the icon will be "orange" to indicate a warning.

Pixel format​

The pixel format configured here will determine the format of the image that is received from the camera. This might not be the pixel format that camera device delivers to the computer, since OneVision will convert the image to the configured format if necessary.

It might be necessary for example to convert an image that is streamed from the camera in Bayer format to BGR format, so that it can be used by the AI models.

Only 3 possible formats are allowed to stream from the camera:

  • Mono8: Monochrome 8-bit
  • Mono16: Monochrome 16-bit
  • BGR8: Blue-Green-Red 8-bit

Consult the supported image formats by images in the Specifications section.

tip

Like OpenCV, OneVision uses the BGR8 format instead of the RGB8 format. This means that the first pixel in the image is the blue component, the second pixel is the green component and the third pixel is the red component.

Rotation​

The rotation of the image can be configured here. The rotation is applied to the image after it is received from the camera, by software. The user will receive the image already rotated.

Rotation can be configured with the following values:

  • 0: The image is not rotated (default).
  • 90: The image is rotated 90 degrees clockwise.
  • 180: The image is rotated 180 degrees clockwise.
  • 270: The image is rotated 270 degrees clockwise.
tip

If the camera's internal features allow rotation and flip, it might be more efficient to configure the rotation directly in the camera device instead of configuring it by software here in OneVision.

Flip​

The flip of the image can be configured here. The flip is applied to the image after it is received from the camera, by software. The user will receive the image already flipped.

Flip can be configured with the following values:

  • None: The image is not flipped (default).
  • Horizontal: The image is flipped horizontally.
  • Vertical: The image is flipped vertically.
  • Both: The image is flipped horizontally and vertically.

Image buffer size​

The image buffer is a ring buffer, which means that it has a fixed size and it is filled with the images that are received from the camera. Once the buffer is full, the oldest images are discarded to make room for the new images.

For many vision applications the image buffer will have no importance, it can be left with the default value.

However, for some applications it might be important to configure the size of the buffer. For example, if the camera and processing work in parallel, not at the same speed, it might be necessary to increase the size of the buffer to avoid losing images if processing is slower than acquisition.

Log level​

This option is used to configure the level of detail of the logs that are generated by the camera controller. This option is mainly intended for developers. The possible values are:

  • DEBUG: The most detailed level of logs.
  • INFO: A normal level of logs (default).
  • WARNING: Only warnings and errors are logged.
  • ERROR: Only errors are logged.
