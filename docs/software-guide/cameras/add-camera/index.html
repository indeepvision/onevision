Add a Camera

Define the cameras that will be used in the project in the section Cameras. Configure here all the features of the cameras and check their status in real time.

Add camera popup​

To add a camera, click on the button in the top left corner.

Controller name​

The name of the camera. Must be unique and is used across the application to identify the camera. It can be changed later.

Controller type​

There are two built-in controller types: GenTL and Emulation.

  • GenTL is a standard controller for GigE Vision cameras.

  • Emulation is a controller that can be used to test the application without a real camera. It generates a synthetic image that can be configured in the Emulation section.

Additional controller types can be added by installing plugins.

Producer​

Once a controller type is selected, multiple producer options might be available, which correspond to different camera manufacturers or different data transmission protocols (transport layer).

  • For the GenTL controller the producers are normally installed with the camera drivers. The producers are DLL files with extension .cti that implement the functionality of the camera for a specific vendor. OneVision will obtain the list of producers from the environment variable GENICAM_GENTL64_PATH (do not confuse it with GENICAM_GENTL32_PATH, for 32-bit applications). This environment variable is normally populated by the camera drivers installer.

  • For the Emulation controller there is only one producer available, which is Emulation.

Vendor​

Read-only field that shows the vendor (or manufacturer) of the producer (not the camera, though in many cases they are the same).

Transport layer​

Read-only field that shows the transport layer of the camera, which is the transmissiong protocol, corresponding to the selected producer. Available Transport layer values are:

Transport layerDescription
GigeGigE Vision (GEV)
Usb3USB3 Vision (U3V)
Usb2USB Video Class (UVC)
EmulationInternal emulation type
UnknownUnrecognized type or various transport layers for the same producer

Device selection​

The device selection sub-menu is used to select the camera device we want to connect the controller to.

Once the camera is connected, the read-only fields in the sub-menu will be populated with information about the camera.

Controller type​

Read-only. The type of controller as indicated in the Add camera popup.

Producer​

Read-only. The producer of the camera as indicated in the Add camera popup.

Vendor​

Read-only. The vendor/manufacturer of the camera is read from the camera itself once it is connected.

Transport layer​

Read-only. The transport layer of the camera is read from the camera itself once it is connected.

Camera serial​

To select the device, enter the in this field with serial number of the camera. The serial number of the camera is normally located on the camera itself.

For Emulation cameras, the serial number is one of the following possibilities: EMU1, EMU2, EMU3, EMU4, EMU5, EMU6, EMU7, EMU8.

Device model​

Read-only. The model is the product identification of the camera established by the manufacturer. It is read from the camera itself once it is connected.

Scan type​

Read-only. The scan type can be one of the following:

  • Areascan: The camera captures the whole image at once.
  • Linescan: The camera captures the image line by line.

This features is read from the camera itself once it is connected.
